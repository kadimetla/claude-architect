# Contributing to CCA-F Crash Course

Thank you for your interest in contributing to this project. Here's how you can help.

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your change (`git checkout -b feat/your-feature`)
3. **Make your changes** following the guidelines below
4. **Test** that all examples and code snippets work correctly
5. **Commit** using [conventional commits](https://www.conventionalcommits.org/) (e.g., `feat:`, `fix:`, `docs:`)
6. **Open a Pull Request** with a clear description of what you changed and why

## Content Guidelines

- All Python code must include docstrings explaining the CCA-F exam concept being demonstrated
- Comments should explain WHY, not WHAT (these are teaching materials)
- Every file header should reference the specific CCA-F Domain and Task Statement it covers
- Use type hints in all Python function signatures
- Error handling must follow the structured error response pattern from Domain 2

## What NOT to Include

- The CCA-F Exam Guide PDF (Anthropic Confidential)
- Actual exam questions or answers
- API keys, tokens, or credentials
- Content from the `private/` directory

## Reporting Issues

Found an error in the study materials? Open an issue with:

- The file and line number
- What you expected vs. what you found
- The relevant CCA-F domain/task statement if applicable
